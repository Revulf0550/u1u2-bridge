Памятка (не перепутать)

  • Pi и ПК НИКОГДА не втыкаются в сам CPE. Только в гнездо LAN PoE-адаптера (или в свитч).
  • Гнездо POE PoE-адаптера → в сам CPE (питание+данные, уже стоит). Гнездо LAN → твоя техника.
  • Конец A: свитч ← Pi-u1 + ПК + LAN от CPE-AP.
  • Конец B: Pi-u2 → LAN от CPE-Client (напрямую, свитч не нужен).
  • Две Pi — на разных концах радио, иначе трафик пойдёт по проводу, а не через мост.
  • AP/Client для данных симметричны (мост прозрачный). Принятая раскладка: AP у u1, Client у u2 — держись её.
  • После переезда: ПК .50 ssh-ит обе Pi по 192.168.1.x — u2 через радио, мост прозрачный.
  • Доступ не теряется: на Pi живут оба адреса (мост .20/.10 + временный домашний DHCP), статика переживает ребут.
