# CTRL-канал — 2-й RS485 переключения каналов VRX

## Статус
**БУДЕМ ДЕЛАТЬ** (с оговорками — см. «Открыто»).

Второй RS485-канал для переключения каналов VRX с u1 был в исходном ТЗ. Сейчас мост
обслуживает только канал управления дроном (CRSF), этого канала ещё нет.

## Суть
Отдельный канал управления с u1 на u2 для переключения каналов видеоприёмника (VRX) на
выносной базе — чтобы оператор с мастер-пульта мог менять частоту/канал VRX дистанционно,
поверх того же транспорта (UDP через CPE710/WireGuard).

## Открыто (нерешённое)
- **В `docs/PLAN.md` отдельного CTRL-канала переключения VRX НЕТ.** Единственное близкое в
  PLAN.md — CRSF back-channel телеметрия RSSI/LQ на пульт Boxer (строки ~132–136), это
  **другое** (телеметрия, не управление VRX).
- **Нужен ли вообще** — открытый вопрос (возможно, переключение VRX решается локально на
  базе/иными средствами).
- **Скорость канала неизвестна** — протокол/бодрейт управления VRX не определён.

## Когда
После определения протокола VRX и решения «нужен ли». Не блокирует текущие этапы.
