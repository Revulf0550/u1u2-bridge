# Обратная телеметрия RSSI/LQ на пульт Boxer

> Трек D. Обратный поток управляющего канала: дать на пульт RadioMaster Boxer
> link-статистику (RSSI/LQ) и предупреждения ELRS о деградации линка.

## Статус

**ЗАПЛАНИРОВАНО — фаза железа. НЕ блокирует текущие этапы.**

Управление дроном работает односторонне (TX-only), failsafe на дроне независим
(ELRS RX уходит в failsafe сам через ~100 мс). Телеметрия на пульт — улучшение
ранней сигнализации деградации линка на дистанции до 1 км, не критичный путь.

Замер 2026-06-04: обратный поток = **0** с обоих концов
(u2 `@elrs` uart→udp = 0; u1 `@p1` udp→uart = 0) при живом прямом RC ~7–33 кБ/с.
Причина **не в коде** — мост симметричный/двунаправленный. Упор физический:
на u2 нет приёмного (RX) пути.

## Суть

Маршрут обратки:
дрон → ELRS-RX → RF → **ELRS-TX модуль (u2)** → CRSF-обратка →
UART7 RX (u2) → мост → `/dev/ttyUSB0` TX (u1) → trainer-вход Boxer.

Link-статистику (RSSI/LQ — кадр CRSF LinkStatistics `0x14`) генерирует **сам
ELRS-TX модуль** и отдаёт «радио» (здесь — u2) по CRSF. То есть RSSI/LQ должны
появиться, как только заработает обратный путь, даже без downlink-телеметрии
от дрона.

Текущая распайка u2: зелёный провод → физ. **pin 26** (UART7_TX, overlay m2) →
SN74HC14N → вход модуля. Только TX. RX-пина в этой конфигурации нет
(на m2 RX = UNCLAIMED, подтверждено живым пинмуксом 2026-06-13).

План (черновой, 4 шага — валидность зависит от первого пункта в «Открыто»):
1. Перевести UART7 с overlay **m2 → m1**: pin **29 = TX**, pin **38 = RX**
   (полный дуплекс на том же UART7 — отдельный второй UART не нужен).
2. Развести **возвратный провод** от телеметрийного выхода ELRS-TX модуля.
3. Поставить **2-й SN74HC14N** на инверсию обратного сигнала во RX (pin 38).
4. Проверить, что переезд TX не сломал управление — **до** подключения RX.

## Открыто (нерешённое)

- **[ГЛАВНЫЙ УЗЕЛ] Half-duplex single-wire vs раздельные TX/RX.**
  CRSF между радио и внешним ELRS-модулем классически **half-duplex по одному
  проводу** (TX и RX мультиплексируются по времени на одной линии). Если у
  конкретного модуля только single-wire CRSF — план из 4 шагов (раздельные
  TX 29 / RX 38 + 2-й инвертор) **невалиден**: понадобится half-duplex-трансивер
  или управление направлением, а не два встречных инвертора. Если модуль даёт
  **раздельные UART TX/RX пады** — полный дуплекс через m1 работает как описано.
  → **определить модель ELRS-TX модуля и его UART-распиновку** — это решает схему.
- **Полярность/уровни обратки.** Подтвердить, что обратный сигнал инвертирован
  так же, как прямой (нужен ли именно SN74HC14N и на какой линии).
- **Приём на стороне Boxer.** Как EdgeTX принимает/показывает CRSF-линкстату
  с trainer-входа — есть ли поддержка и в каком виде RSSI/LQ доходят до экрана.
- **Риск переезда m2→m1.** UART7 несёт РАБОЧЕЕ управление; смена overlay —
  потенциально ломающая операция. Нужен план отката на m2/pin 26.

## Когда

Триггер: после полевых испытаний текущего канала, либо когда понадобится ранняя
сигнализация деградации линка на 1 км. Делается в 2 фазы, **обе — за
drone-safety gate** (трогаем канал управления): сначала переезд UART7 m2→m1 и
проверка TX без дрона, затем разводка RX + 2-й инвертор + бинд-тест.
**Не блокирует** текущие этапы.
