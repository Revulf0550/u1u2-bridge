# Обратная телеметрия RSSI/LQ на пульт Boxer

> Трек D. Обратный поток управляющего канала: дать на пульт RadioMaster Boxer
> link-статистику (RSSI/LQ) и предупреждения ELRS о деградации линка.

## Статус

**ОТЛОЖЕНО → доработки (backlog). Главный технический узел РЕШЁН (2026-06-14).**

НЕ блокирует текущие этапы. Управление работает односторонне (TX-only), failsafe
на дроне независим (ELRS RX уходит сам через ~100 мс). Возобновлять — по плану
«Когда» (3 фазы, последняя за drone-safety gate).

Модуль (уточнён): **RadioMaster Ranger Micro** (2.4 ГГц, JR-micro bay).

Замер 2026-06-04: обратный поток = **0** с обоих концов (u2 `@elrs` uart→udp = 0;
u1 `@p1` udp→uart = 0) при живом прямом RC ~7–33 кБ/с. Причина **не в коде** —
мост симметричный/двунаправленный. Упор физический: на u2 нет приёмного пути.

## Решено (2026-06-14): half-duplex подтверждён, full-duplex план ОТМЕНЁН

Источники (ExpressLRS docs + GitHub discussions): CRSF между радио и внешним
ELRS-модулем через JR-bay идёт по пину **S.Port, ОДНИМ проводом, инвертированный
half-duplex** — TX и RX мультиплексируются по времени на одной линии. Отдельного
выхода телеметрии у модуля (через bay) НЕТ.

Следствия:
- **Full-duplex план ОТМЕНЁН.** Старое «TX 29 / RX 38 + 2-й инвертор на отдельный
  пин модуля» невалидно: отдельного пина нет, и команды, и телеметрия идут по
  одному проводу (том же зелёном, что уже в pin 26).
- **«2-й SN74HC14N» НЕ нужен.** SN74HC14N — HEX (6 гейтов), управление занимает 1.
  Half-duplex-узлу нужно 2 гейта — оба из существующего чипа.

## Кандидат-узел (на стенд, НЕ финал)

Half-duplex на одном проводе, целиком на имеющемся железе:
- **U1** (текущий управляющий гейт SN74HC14N): OPi TX (pin 29) → инверсия →
  **R ~1 кОм последовательно** → общая линия.
- **U2** (добавленный гейт того же SN74HC14N): общая линия → инверсия →
  OPi RX (pin 38). Вход гейта высокоомный — слушает линию.
- **R** делает удержание линии со стороны OPi мягким → модуль перебивает её,
  когда шлёт обратку. (open-drain + pull-up «как S.Port» здесь не подходит:
  линия инвертированная, в покое «0»; практичный эквивалент — push-pull + R.)
- **UART-режим:** обычный full-duplex на m1 (29/38). Спец-режим single-wire НЕ
  нужен — слияние в один провод делает железо.
- **Софт (правка `crsf_bridge.py`):** RX слышит собственное эхо TX. На
  направлении u2→u1 пропускать только кадры телеметрии (LinkStatistics `0x14`
  и т.п.), RC-эхо (`0x16`) отбрасывать. Насквозь-прозрачным мост остаётся
  только в одну сторону.

R (номинал) и эхо-фильтр валидировать на стенде.

## Что это даёт (область ценности)

В основном **RSSI/LQ именно канала управления + аудио-предупреждения ELRS на
Boxer** («telemetry lost» / critical RSSI). Ранний сигнал деградации командного
линка на 1 км — видео-OSD его не покажет (каналы управления и видео независимы).
Полётная телеметрия (GPS/батарея/высота) — бонус и, вероятно, уже есть в OSD по
видео; требует телеметрийного провода полётник→RX + настройки.

- ⚠ Таблица распайки в `docs/wiring-opi5max.md` (pin 29/38, раздельные TX/RX) —
  на ОТМЕНЁННОМ full-duplex допущении. Не паять по ней; при возобновлении
  переписать под half-duplex (один провод).

## Когда (план возобновления — 3 фазы)

1. **Наблюдение (0 изменений в боевом).** Высокоомным тапом проверить, что модуль
   шлёт обратку (LinkStatistics) по линии. Не шлёт — копать настройку ELRS.
2. **Стенд (обратимо).** Собрать узел (U1 + U2 + R) на запасных пинах, доказать,
   что RSSI/LQ доходят в OPi. Боевой канал управления при этом не трогаем.
3. **Коммит (необратимо).** Перепайка m2→m1 + проверка на Boxer. **Drone-safety
   gate.** Только после стендового подтверждения.

Триггер: после полевых текущего канала, либо когда нужна ранняя сигнализация
деградации линка на 1 км.
