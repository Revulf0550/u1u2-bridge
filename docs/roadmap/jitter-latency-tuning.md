# Тюнинг jitter-buffer latency

## Статус
**БУДЕМ ДЕЛАТЬ на полевых.** На столе джиттера нет — тюнить негде.

## Суть
Опустить `rtpjitterbuffer latency` в `u1/video_rx.sh` (env `JITTER_LATENCY`) с текущих
~50 мс к ~15–20 мс под реальным радио (CPE710 PtP).

- Радио-джиттер измерен ~2 мс — буфер 50 мс избыточен.
- Дропы кадров начинались примерно с ~10 мс буфера — это нижняя граница, ниже не уходить.
- Выигрыш: ~30 мс glass-to-glass видео-латенции.

Это основной knob latency на приёмной стороне (см. CLAUDE.md, раздел GStreamer:
`rtpjitterbuffer latency=<env> drop-on-latency=false do-lost=true`).

## Когда
На полевых испытаниях, под реальным радио CPE710. На столе (bench) джиттера нет,
подбирать значение бессмысленно.
