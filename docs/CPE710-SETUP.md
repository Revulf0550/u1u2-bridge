# Настройка TP-Link CPE710 — PtP-мост 5 ГГц

Делается **один раз на пару устройств**, до или параллельно деплою Orange Pi.
После настройки эта пара работает как «прозрачный кабель» между двумя
Pi: оба устройства видят друг друга в одной L2-сети `192.168.1.0/24`,
без какого-либо роутинга.

> **Почему именно эти параметры:** приоритеты проекта — надёжность канала
> и низкий джиттер видео. 20 МГц канал устойчивее в движении, чем 40/80;
> фиксированный канал 149 (UNII-3, без DFS) убирает риск переключения
> канала во время сеанса; bridge-режим вместо router снимает NAT-overhead;
> отключённый MAXtream TDMA даёт меньшую латентность при одном клиенте.

---

## 0. Что вам понадобится

- 2× TP-Link CPE710 (одной модели/прошивки).
- 2× PoE-инжектора из комплекта + 2× блока питания.
- Ноутбук с Ethernet-портом и любым браузером.
- Один LAN-кабель для соединения двух CPE напрямую — для финальной
  проверки радиоканала.

---

## 1. Первый вход в Pharos UI

1. Подключите CPE через PoE-инжектор: LAN-порт инжектора (он же `LAN0`) — в
   ноутбук, PoE-порт — в CPE.
2. Дайте CPE 30–60 секунд на загрузку.
3. На ноутбуке настройте статический IP в подсети CPE по умолчанию:
   `192.168.0.10/24`, шлюз пустой.
4. Откройте `http://192.168.0.254` в браузере. Логин/пароль по умолчанию:
   `admin` / `admin`. UI попросит сменить пароль — поставьте свой и
   запишите.
5. UI = **Pharos OS** (TP-Link). Дальнейшие шаги — внутри неё.

Повторите для второго CPE.

---

## 2. Конфигурация — master (Access Point)

Один из двух CPE будет точкой доступа (AP), второй — клиентом (Client).
Решите сейчас и физически промаркируйте корпуса (стикер «AP» / «Client»),
дальше пути расходятся.

В UI **первого** CPE:

| Раздел                          | Параметр                | Значение                      |
| ------------------------------- | ----------------------- | ----------------------------- |
| **Quick Setup → Operation Mode** | Mode                    | **Access Point** (часть Bridge — НЕ Router) |
| Wireless → Wireless Mode        | Mode                    | **Access Point**              |
| Wireless → SSID                 | SSID                    | `u1u2-bridge` (или любой)     |
| Wireless → Security             | Mode                    | **WPA2-PSK**                  |
| Wireless → Security             | PSK                     | (длинный, запишите)           |
| Wireless → Channel              | Channel                 | **149** (фиксированный)       |
| Wireless → Channel Width        | Width                   | **20 MHz** (НЕ 40, НЕ 80)     |
| Wireless → Transmit Power       | Power                   | **17–20 dBm**                 |
| Wireless Advanced → Distance    | Distance                | **1 km** (или ваша реальная)  |
| Wireless Advanced → MAXtream    | MAXtream TDMA           | **Disable**                   |
| Network → LAN                   | IP                      | `192.168.1.2/24`              |
| Network → LAN                   | DHCP Server             | **Disable**                   |

> **Почему именно 17–20 dBm, а не 27 (max):** полная мощность на коротких
> дистанциях вызывает desensitization приёмника на другом конце —
> сигнал «оглушает» front-end, и эффективное SNR ПАДАЕТ. Для дистанции
> 1 км 17–20 dBm даёт лучший link budget.

> **Почему 149, а не auto:** auto может переключиться на DFS-канал
> (52–144), где CPE по нормам должен ждать 60 секунд при обнаружении
> радара — это разрыв на минуту посреди полёта. UNII-3 (149–165) — без
> DFS.

Сохранить, дождаться перезагрузки CPE.

---

## 3. Конфигурация — slave (Client)

В UI **второго** CPE:

| Раздел                          | Параметр                | Значение                         |
| ------------------------------- | ----------------------- | -------------------------------- |
| **Quick Setup → Operation Mode** | Mode                    | **Client** (часть Bridge)        |
| Wireless → Wireless Mode        | Mode                    | **Client**                       |
| Wireless → SSID                 | Lock to AP MAC          | MAC адрес master CPE (из его UI → Status → Wireless) |
| Wireless → SSID                 | SSID                    | тот же `u1u2-bridge`             |
| Wireless → Security             | PSK                     | тот же PSK что на master         |
| Wireless → Channel Width        | Width                   | **20 MHz** (как у master)        |
| Wireless → Transmit Power       | Power                   | **17–20 dBm**                    |
| Wireless Advanced → Distance    | Distance                | **1 km**                         |
| Wireless Advanced → MAXtream    | MAXtream TDMA           | **Disable**                      |
| Network → LAN                   | IP                      | `192.168.1.3/24`                 |
| Network → LAN                   | DHCP Server             | **Disable**                      |

> **Lock to AP MAC** — критично. Без него Client может зацепиться за
> любой CPE с тем же SSID в радиусе (актуально на тренировочных полях
> с несколькими командами). С привязкой по MAC — только за свой master.

Сохранить, дождаться перезагрузки.

---

## 4. Проверка radio-link

В UI master CPE → **Status → Wireless**:

- **Client List** должен показать MAC slave-CPE.
- **Tx/Rx Rate** — ненулевые, обычно 100–300 Mbps на 20 МГц.
- **SNR** — > 25 dB на бенче (≤ 10 м), > 15 dB на полевой дистанции.

В UI slave CPE → **Status → Wireless**:

- **AP MAC** = MAC master.
- **SNR** аналогично.

Если SNR < 10 или линк не устанавливается — частые причины:
- Антенны не направлены друг на друга (CPE710 — narrow-beam 9°×9°).
- Канал 149 занят соседями (проверь Site Survey, переключи на 153/157/161).
- PSK или SSID не совпадают.

---

## 5. Контрольная проверка через прямой LAN-кабель

Цель — убедиться, что радиоканал РЕАЛЬНО пропускает Ethernet-трафик,
ДО подключения Orange Pi. Это снимает один источник неопределённости
при последующей отладке.

1. Подключите ноутбук кабелем к LAN-порту PoE-инжектора master CPE.
2. На ноутбуке — статический IP `192.168.1.100/24`, gateway пустой.
3. Подключите второй ноутбук (или ту же машину, переключив кабель)
   аналогично к slave CPE с IP `192.168.1.101/24`.
4. С первого ноутбука: `ping 192.168.1.101`.

   Ожидается: устойчивые ответы 5–15 мс на бенче, потери 0%.

5. Если есть только один ноутбук — пингуйте `192.168.1.2` (master LAN)
   и `192.168.1.3` (slave LAN) поочерёдно с одной стороны. Оба IP
   должны отвечать.

Если ping не идёт — проблема в Pharos UI, **не** в Orange Pi и **не**
в нашем коде. Возвращайтесь к §2–§4 и проверяйте параметры заново.

---

## 6. Дальше

После того как radio-link подтверждён:

- Не сбрасывайте конфигурацию CPE — она нужна для полёта.
- Запишите итоговые параметры (SSID, PSK, MAC-адреса, канал) в локальный
  passwords manager или физический блокнот. Восстанавливать с нуля — 30 минут.
- Переходите к `docs/DEPLOYMENT.md` §2 — подключение Orange Pi к этим
  же LAN-портам PoE-инжекторов.
