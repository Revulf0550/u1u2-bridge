## §3 — Рабочие инварианты (перенос, не менять)

- Язык — русский.
- SSH всегда `ssh -i ~/.ssh/u1u2 ubuntu@<ip>`.
- Drone-safety gate: пропеллеры сняты + Boxer off перед любым рестартом CRSF-сервисов;
  явное разрешение оператора на каждое питание дрона.
- Два канала: Claude Code (WG on, Режим 1, репо/git/SSH) vs батч-PowerShell-транскрипт
  (WG off, Режим 2, доступ к мосту 192.168.1.x). Переключение инструментов — анонсировать.
- **Режим 2 (direct) достижим ТОЛЬКО при WG off — для ARDOR.** Причина: kill-switch
  `AllowedIPs=0.0.0.0/0` заворачивает весь трафик ПК в туннель, параллельного прямого
  доступа к мосту нет. (Сам конфиг `AllowedIPs` в этой сессии живой командой не
  перепроверялся — факт из памяти; поведение «WG on → нет прямого доступа с ПК» с ним
  согласуется.) На узлах wg0 может быть active одновременно с Режимом 2 — мост идёт по
  192.168.1.10 мимо wg0, доставку не ломает (наблюдение §5 хэндоффа 06-24).
- Серийный ping (`-n 4`+) обязателен за радио — одиночный врёт на холодном ARP.
- §0-аудит (git + live-сервисы) ПЕРЕД любыми мутациями/хэндоффом. Транспорт — по `PEER=`
  в env, НЕ по `systemctl is-active`.
- Scoped-коммиты: НИКОГДА `git add -A`; diff-review перед каждым коммитом; форвард-онли.
- Файлы хэндоффа/скриптов — генерировать на стороне Claude и отдавать скачиванием, не
  heredoc (порча при вставке в терминал). Claude Code копирует из Downloads в репо.
