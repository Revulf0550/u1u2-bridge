## СОСТОЯНИЕ (факт, сверено вживую 2026-06-05)

**2b — физический cutover ВЫПОЛНЕН:**
- u1 → мини-свитч → PoE-адаптер CPE-AP (`LAN`); в тот же мини-свитч — ARDOR `Ethernet 3`.
- u2 → PoE-адаптер CPE-Client (`LAN`).
- CPE-коробки — в порты `POE` своих адаптеров. CPE-AP `192.168.1.2`, CPE-Client `192.168.1.3`.
- Обе Pi на `192.168.1.20/.10`, `192.168.31.x` пропал (реально ушли с домашнего свитча).

**2c — проверка по радио ВЫПОЛНЕНА, включая live:**
- **CRSF u1→u2 через радио:** u1 шлёт `uart->udp ≈ 17.5 KB/s`, u2 принимает `udp->uart ≈ 17.7 KB/s`, `udp_drop=0` на обоих — лосслесс, симметрично.
- **Видео u2→u1 через радио:** идёт (на no-signal битрейт скромный ~600 кбит — CBR не добивает на статике; это ожидаемо).
- **LIVE С ДРОНОМ (подтвердил пользователь):** дрон реагирует на стики (управление Boxer→дрон через радио), картинка живая на мониторе u1. Сквозной тракт подтверждён по-настоящему.
- **Радио:** 0% потерь, ~2 мс (15/15 и 10/10 пингов).
- SSH на обе Pi с моста работает (после правки UFW на u2, см. ниже).

## УРОКИ СЕССИИ (внести в CLAUDE.md)
1. **WG kill-switch режит локалку.** WireGuard на ARDOR с `AllowedIPs=0.0.0.0/0` блокирует весь не-туннельный трафик (включая `192.168.1.x`) → ping/ssh «Общий сбой» при поднятом интерфейсе и валидном IP. **Правило:** для работы с локальным мостом — WG деактивировать; «Общий сбой» (не таймаут) = локальный фильтр/маршрут, не сеть.
2. **UFW + смена источника = тихий дроп (повтор, теперь на SSH).** После cutover ARDOR пошёл к u2 с нового `192.168.1.50`; u2 UFW его не пускал (allow были только под `10.8.0.x`/`192.168.31.x`) → SSH timeout при живом ping. **Правило:** при смене источника открывать UFW на приёмнике под новый source для КАЖДОГО нужного порта (не только данные, но и 22).
3. **Откат в рабочий режим может иметь физический префикс.** После физического cutover откат в WG-режим требует сперва вернуть кабели Pi в сеть с интернетом — иначе WG не встанет. **Правило:** в процедуре отката проверять транспортные предпосылки (есть ли у узла путь к endpoint), не только конфиг.
4. **Двойной NIC решает dual-home без footgun.** На ARDOR второй (USB) NIC под мост + встроенный под интернет = чистое разделение, без secondary-IP на одном адаптере (который рвёт интернет).
5. **Не верить маленькому tcpdump-сэмплу** (перенос из сессии 2a — в CLAUDE.md ещё НЕ внесён). `-c 10` за ~100 мс попал в межкадровый промежуток → ложная тревога «видео мёртвое». **Правило:** оценка RTP — счёт пакетов / гистограмма размеров за ≥5 c, не `-c 10`.
6. **CBR маскирует no-signal** (перенос из 2a). Поток ~2.5 Mbps идёт и на статике; здоровый поток ≠ живой контент. Сегодня закрыто live-проверкой с дроном. **Правило:** «живой контент» подтверждать движением/дроном, не фактом наличия RTP.
