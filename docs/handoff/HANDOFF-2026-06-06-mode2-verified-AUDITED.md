## 🔌 СВИТЧ НА КОНЦЕ A — ОБЯЗАТЕЛЕН (находка этой сессии)

u1 **не поднимает линк** напрямую в `LAN` PoE-адаптера CPE-AP — **даже с заведомо рабочими кабелями** (доказано перебором). При этом ARDOR линкуется напрямую в CPE-AP, а u2 — в CPE-Client. → специфическая несовместимость авто-нег сетевухи u1 (RK3588 GMAC) ↔ CPE-AP через пассивный PoE. **Мини-свитч решает** (активный партнёр по линку) и заодно делит один `LAN`-порт между u1 и ARDOR.

**Раскладка Конца A (держать постоянно, в поле тоже):**
```
CPE-AP ──[POE]адаптер[LAN]── мини-свитч ──┬── Pi-u1
                                          └── ARDOR (Ethernet 3)
```
Конец B: `Pi-u2 → LAN адаптера CPE-Client` напрямую (свитч не нужен). Свитч запитать (5V/USB).
> ⚠️ Техника — ТОЛЬКО в порт `LAN` адаптера. В `POE`/в сам CPE — никогда (пассивное напряжение спалит NIC).

## 🔑 ДОСТУП И СЕТЬ ARDOR (verified)

- **ARDOR — два проводных NIC:** встроенный `Ethernet` (Realtek PCIe GbE) = `192.168.31.150` (дом, **интернет/ассистент**); USB-донгл `Ethernet 3` (Realtek USB 2.5GbE) = `192.168.1.50/24` **без шлюза** (мост). WG-адаптер `NSU-pc` = `10.8.0.5`.
- **Интернет ARDOR идёт по встроенному `Ethernet` и от WG не зависит** → можно работать вживую с выключенным WG (батч/транскрипт не обязателен).
- **WG kill-switch (`AllowedIPs 0.0.0.0/0`) режет всю `192.168.1.x`** → для моста WG **выкл**. Симптом «Общий сбой» (не таймаут) при живом IP = локальный фильтр WG, а не сеть.
- **Доступ к мосту:** `ssh -i ~/.ssh/u1u2 ubuntu@192.168.1.20` (u1) / `@192.168.1.10` (u2), sudo NOPASSWD. В PowerShell ключ — `$HOME\.ssh\u1u2`.
- **NB про донгл:** `Ethernet 3` при выдёргивании из USB **пропадает из `ipconfig`/`Get-NetAdapter` целиком** (встроенный показал бы «отключён»). Нет связи с мостом → первым делом проверить, что донгл воткнут и на нём `.50`.

## ⚠️ УРОКИ / ИНЦИДЕНТЫ (чтобы не повторять)

> Лейтмотив: почти все «отказы» — провода/USB-контакт, не софт. Проверять физику ПЕРВОЙ (`lsusb`/`ls /dev` на Pi, `Get-NetAdapter` на ARDOR, лампочки линка).

1. **u1 CH340 не определялся** (`dmesg error -71`) — плохой USB-переходник; лечение: прямое/хорошее подключение → `/dev/ttyUSB0`.
2. **ARDOR не видел мост** — донгл `Ethernet 3` не воткнут (пропал из ipconfig).
3. **u2 `video-tx` crash-loop (471×)** — `/dev/video0` отсутствовал; лечение: подключить грабер.
4. **u1 «недоступен, но шлёт CRSF»** — полу-битый Ethernet при перетыках на Конце A.
5. **u1 не линкуется напрямую в CPE-AP** — нужен свитч (см. выше).
6. **UFW + смена источника = тихий дроп.** После cutover ARDOR идёт с `192.168.1.50`; если UFW на Pi не allow этот source per-port (вкл. **22**) → SSH timeout при живом ping. Правила добавлены на cutover; при повторе — проверить UFW source.

## ⛔ НЕ ДЕЛАЙ (carry-over)

- **НЕ редеплоить `install.sh` без `TRANSPORT=direct SKIP_NETPLAN=1`** — иначе CRSF-env вернётся к `10.8.0.x` и перезапишется netplan.
- **НЕ `kmssink`** (VOP2 timeout) — только `cage`+`waylandsink`. **НЕ убирать `videoconvert`** перед `waylandsink` (чёрный на NV12). Грабер — только **640×480 MJPG@30, без `io-mode=4`**.
- **WG на ARDOR держать ВЫКЛ при работе с мостом** (kill-switch). Включать только для Режима №1/отката.
- **Pi на разных концах радио** в Режиме №2 (иначе трафик уйдёт по проводу).
- **ДРОН/CRSF трогать — ТОЛЬКО спросив пользователя ЯВНО.**
- Не добавлять secondary-IP на встроенный Ethernet ARDOR (рвёт интернет) — мост на отдельном NIC.
- Claude Code Bash: без `cd ~/...`. Длинные файлы — через present_files, не вставкой в терминал.
