### КЛЮЧЕВАЯ НАХОДКА — живой видео-тест
Дрон включали (Режим 1 / интернет; **CPE710 НЕ задействован**; пропеллеры сняты, Boxer выключен).
- **Видео-дропы — АНАЛОГОВЫЕ**, в линке дрон-VTX → VRX (5.8 ГГц), **выше** цифрового моста. Мост здоров — честно несёт то, что отдаёт VRX.
- **Снег** = перегрузка приёмника из-за близости антенн VTX/VRX → ушёл после разнесения / снижения мощности VTX.
- **Синий экран** = аналоговый VRX теряет захват сигнала (no-signal blue), **НЕ цифровая потеря** (та замораживает кадр, синий дать не может). Сейчас остаётся **периодический синий** — вероятно, мощность VTX занижена после ручного снижения; нужна «середина» (не снег / не синий).
- Механизм: **порог синхронизации**, «всё или ничего» — выше порога проходит зашумлённый кадр; ниже — VRX/грабер гасит в синий.
- **Вывод: FEC / GOP / энкодер синий НЕ лечат** — он аналоговый, до моста. Лечится только на RF/аналог-уровне.

### Решение по граберу
- **Текущий:** Arkmicro `18ec:5555` (EasyCAP-клон), 640×480 MJPG.
- **Выбран апгрейд:** **cx231xx (Hauppauge USB-Live2)** — мейнлайн-драйвер Linux, YUYV 4:2:2, 720×576 PAL, отдаёт статус сигнала (`no signal / no hsync`), «честный оцифровщик» (пропускает шум, а не гасит). ~€45 (Conrad/eBay.de; NL: maxict.nl, proshop.nl). **Заказывается.**
- **ОГОВОРКА:** грабер снимет синий **только если гасит грабер, а не VRX** — **НЕ проверено** (тест «VRX vs грабер: кто гасит первым» не делали). Как апгрейд качества — оправдан в любом случае.
- **Черновик правки `video_tx.sh`** под cx231xx (НЕ применён):
  - убрать `image/jpeg ! jpegdec` → `video/x-raw,format=YUY2,width=720,height=576,framerate=25/1`
  - добавить `deinterlace method=linear` (композит чересстрочный); для мин. латенции — одно поле `fields=top`
  - `v4l2src norm=PAL` (NTSC → `norm=NTSC`, 720×480@30)
  - энкодер (`mpph264enc …`) — без изменений
  - применять через Claude Code с diff-review, **после** `v4l2-ctl --list-formats-ext -d /dev/video0` на живом грабере.
