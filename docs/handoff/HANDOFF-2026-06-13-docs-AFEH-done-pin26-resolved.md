**Сага pin 26 — закрыта.** Live `gpio readall` на боевой u2 (2026-06-13): физ. **pin 26 = GPIO1_B5 = UART7_TX overlay m2 (ALT10) → `/dev/ttyS7`**, TX-управление односторонне. Пользователь был прав; «pin 24 / 29» из старых снапшотов — мимо. Пины 19/21/23 (SPI0) — простой GPIO IN, т.е. ALT10 на 24/26 это именно UART7-m2, не SPI. m1 (29/38) неактивен — резерв под RX/телеметрию (трек D). Четыре дока единогласны.

## ЖЕЛЕЗО / ИСТИНА (зафиксировано)
- **u2** (drone, headless): зелёный провод → физ. **pin 26 = UART7_TX m2 → `/dev/ttyS7`**, TX-only, через SN74HC14N. RX/телеметрия = m1 (29/38), неактивно. Грабер Arkmicro `18ec:5555` → `/dev/video0`. IP: `10.8.0.7` (туннель) / `192.168.1.10` (мост).
- **u1** (оператор): CH340 `1a86:7523` → `/dev/ttyUSB0` (CRSF от RadioMaster Boxer). HDMI-монитор. IP: `10.8.0.6` / `192.168.1.20`.
- **CPE710**: AP `192.168.1.2`, Client `192.168.1.3`, ch 149, 20 МГц, TDMA off, 1 км.
- **Топология (locked):** CPE-AP → LAN PoE-адаптера → **мини-свитч** → {u1 + ПК}; u2 → LAN CPE-Client напрямую. **Мини-свитч на Конце A обязателен** (GMAC RK3588 не линкуется напрямую через пассивный PoE). Технику — только в LAN адаптера, никогда в POE/в сам CPE.
- **Каналы:** 1× ELRS-управление (TX-only) + 1× видео. (Не «2 ELRS» — это устаревшее из HANDOFF.md.)
- **Сервисы** (посл. подтв. 2026-06-06): u1 — `crsf-bridge@p1`, `video-rx`, `wg-quick@wg0`; u2 — `crsf-bridge@elrs`, `video-tx`, `wg-quick@wg0`.
