## 1. ВЫПОЛНЕНО (сессия 2026-06-04)

**Задача сессии — снизить задержку видео — закрыта. Главный выигрыш: −450 мс
от джиттербуфера.**

- **SSH-ключи** на обе Pi — беспарольный вход (`~/.ssh/u1u2`), беспарольный sudo.
- **Скрипты задеплоены в `~`** на каждой Pi: `u1:~/video_rx.sh`, `u2:~/video_tx.sh`
  (LF, shebang чистый). Гоняли из `~`, НЕ из `/opt`.
- **`/run/user/0` создан вручную на u1** (был отсутствует — tmpfiles из `install.sh`
  не развёрнут). tmpfs → пропадёт при ребуте.
- **Baseline на `videotestsrc`:** egress(u2)=ingress(u1)=254 пкт/8с, **0 потерь**.
  Парадокс Этапа 2 («0 пакетов на wg0») НЕ воспроизвёлся — потому что и репо-скрипт,
  и ручная команда без сломанного `profile=baseline` capsfilter (старая `/opt`-копия
  на u2 его содержит — см. §5).
- **Подбор `JITTER_LATENCY`** (на мониторе u1, паттерн «ball»): плавно вплоть до
  20 мс, **порог дропов ~10 мс** (лёгкие подтормаживания). Принят **дефолт 50 мс**
  (5× запас, в 10 раз меньше прежнего 500).
- **Микро-выигрыш `videoconvert` ОТКЛОНЁН:** без него на live — чёрный экран,
  `waylandsink` не ест NV12 напрямую (согласует только RGBx). Записано в `CLAUDE.md`.
- **Живой e2e на камере дрона:** живая картинка, **лаг небольшой**.
- **Коммиты (запушены в `origin/main`):**
  - `3bbafc9` — правило §0 в `CLAUDE.md` (`## Process`).
  - `0befafb` — `video_rx.sh` дефолт `JITTER_LATENCY` 500→50 + урок про NV12.
- **Факт после сессии:** `video_rx.sh:21` = `JITTER_LATENCY:-50`; `main...origin/main`
  синхронизировано.

## 2. СЛЕДУЮЩЕЕ (приоритет сверху вниз)

1. **Этап 3 — CPE710 PtP** (главный остаток задержки): уберёт ~90 мс интернет-WG.
   После разворачивания **`JITTER_LATENCY` подбирать заново** — число link-specific;
   на коротком линке (3–7 мс) стартовать с 20–30, порог искать так же на мониторе.
   `MTU` в `video_tx.sh` **пересмотреть под новый линк** (1200 был под wg-интернет;
   проверить path MTU CPE710). Снять `SKIP_NETPLAN`/`PEER_IP_OVERRIDE`.
2. **Задеплоить в systemd через `install.sh`** на обе Pi. ⚠️ Сейчас рабочие скрипты
   только в `~`; в `/opt/u1u2-bridge/u2/video_tx.sh` лежит **СТАРАЯ СЛОМАННАЯ** копия
   (`io-mode=4`, `profile=baseline` capsfilter, `mtu=1400`) — без редеплоя она
   воскресит баг «0 пакетов». `sudo ./install.sh u2` (и `u1`) перезапишет.
3. **cage под systemd** (`video-rx.service`) на железе НЕ проверен — нетривиально
   (владение TTY у root). Гоняли руками: `sudo env JITTER_LATENCY=50 bash ~/video_rx.sh`.
4. **`video.env`** `install.sh` пока НЕ генерирует — дефолты живут в скриптах.

## 3. Топология

- **u2 = TX:** `ssh -i ~/.ssh/u1u2 ubuntu@10.8.0.7` (или пароль `ubuntu`).
- **u1 = RX + HDMI-монитор:** `ssh -i ~/.ssh/u1u2 ubuntu@10.8.0.6`.
- Транспорт: WireGuard `10.8.0.x` через интернет, RTT ~150 мс, iface `enP3p49s0`.
  CPE710 **НЕ развёрнут** — Этап 3.
- CRSF `crsf-bridge@tx2.service` на u2 — **active**, в этой сессии не трогали.

## 4. Рабочая конфигурация (проверена на железе; истина — скрипты в репо)

**TX (u2), живой грабер:** `bash ~/video_tx.sh` (Arkmicro 640×480 MJPG → mpph264enc
baseline → RTP mtu=1200 → udpsink 10.8.0.6:5600). Для дебага без дрона — заменить
`v4l2src` на `videotestsrc is-live=true pattern=ball ! video/x-raw,format=NV12,width=640,height=480,framerate=30/1`.

**RX (u1), под cage с sudo:**
```
sudo env JITTER_LATENCY=50 bash ~/video_rx.sh
```
(скрипт сам делает `env XDG_RUNTIME_DIR=/run/user/0 cage -- gst-launch … rtpjitterbuffer
latency=50 … mppvideodec ! videoconvert ! waylandsink sync=false`). `/run/user/0` на
tmpfs — после ребута создать заново: `sudo mkdir -p /run/user/0 && sudo chmod 0700 /run/user/0`.

## 5. НЕ ДЕЛАЙ

- **НЕ возвращай `kmssink`** — сломан на этой плате (VOP2: `wait pd0 off timeout`,
  `unexpected power on pd5`). Дисплей жёстко на `cage + waylandsink`.
- **НЕ убирай `videoconvert`** перед `waylandsink` — чёрный экран (NV12 не согласуется).
- **НЕ запускай `/opt/.../video_tx.sh` на u2** — старая сломанная копия. Истина в репо/`~`.
- **`pyproject.toml`** — есть незакоммиченное чужое изменение (ruff/mypy + `u1`),
  оставлено нетронутым, не наше; в коммиты сессии не включалось.

## 6. Измерение (без таймера)

- **Тюнинг дропов — на `videotestsrc`** (дрон не нужен). Движущийся паттерн «ball»
  на мониторе u1 — дропы/рывки видно сразу. Перед картинкой — `tcpdump -i wg0 'udp
