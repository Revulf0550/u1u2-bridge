## §3 — switch-mode.ps1: подтверждённая сильная инженерия (перенос знания)

Прочитан целиком, корректен и безопасен:
- **Preflight-ping обеих Pi** (`Test-Connection -Count 2 -Quiet`) ПЕРЕД любой мутацией → `throw`
  при недоступности (lockout-защита: битый конфиг не уедет на недоступную Pi). Серийный, не одиночный.
- **Деплой через `git archive HEAD`** — на Pi уезжает зафиксированный HEAD, не рабочее дерево.
- **Rebuild не патч:** `rm -rf ~/u1u2-deploy && tar -xf` каждый раз с нуля.
- **Drone-safety:** рестарт сервисов под явным `[y/N]` + предупреждение «дрон выключен/на земле».
  `-NoRestart` = деплой конфига без разрыва CRSF.
- **netplan НАМЕРЕННО не трогается:** сосуществуют статика 192.168.1.x (`99-u1u2-bridge.yaml`)
  и cloud-init DHCP (`50-cloud-init.yaml`), location-adaptive (мост=статика, дом=DHCP для WG).
- Карта режимов: tunnel → u1=10.8.0.6 / u2=10.8.0.7 (WG on); direct → u1=192.168.1.20 /
  u2=192.168.1.10 (WG off). Сервисы `crsf-bridge@p1`(u1)/`crsf-bridge@elrs`(u2) + video-rx/tx.
- Мелочь (P3): финальная проверка через `systemctl is-active` — не показывает transport-режим
  (нужен `PEER=`), но здесь режим уже задан через `TRANSPORT=`, для «поднялся ли» хватает.
