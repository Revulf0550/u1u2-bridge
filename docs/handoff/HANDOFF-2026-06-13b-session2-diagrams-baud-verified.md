## pin 26 — ПОДТВЕРЖДЁН (закрыто, не переоткрывать)

*(источник: эмпирика — дрон биндится/управляется через `/dev/ttyS7` при проводе на pin 26; план пользователя)*

- **Физ. pin 26 = боевой UART7_TX (m2) → `/dev/ttyS7`. Работает, подтверждено. НЕ переоткрывать.**
- **Пины 29/38 = m1 — резерв под телеметрию (трек D).** По плану, не трогать.
- Нюанс на будущее (не блокер, делать необязательно): `pi-u1/u2-board-pinout.svg` печатают на pin 26 *стоковую* функцию чипа из даташита (UART1_TX_M1) — это легитимный чип-референс, а деплойный UART7 показан зелёной пометкой «сигнал → инвертор» на том же pin 26. Если когда-нибудь захочется — можно добавить однострочную пометку «деплой: UART7 (overlay m2)» на pin 26. Сам pin 26 при этом закрыт.

## ОСТАЛОСЬ / ТРЕКИ — *(источник: поиск по чатам + хэндофф 2026-06-13a)*

### Трек C/D — телеметрия RSSI/LQ на пульт Boxer (**ЖЕЛЕЗО + drone-safety gate**)
Обратный поток = 0. Путь: дрон → ELRS-TX (u2) → ttyS7 → мост → ttyUSB0 (u1) → trainer Boxer. Аппаратный упор: на m2/pin 26 RX не поднимается (UNCLAIMED). Нужно: (1) перевести UART7 с m2 на **m1 (TX 29 / RX 38)**; (2) возвратный провод от телеметрийного выхода ELRS; (3) второй гейт **SN74HC14N** на инверсию обратки в RX (pin 38); (4) понять, как ELRS отдаёт телеметрию (CRSF half-duplex vs раздельные TX/RX). Мост уже двунаправленный. Завести спеку в `docs/roadmap/`. Требует drone-safety gate и перепайки.

### Трек B — исследование дропов видео (**research, без железа**)
Гипотеза: сыпется по краям, центр стабилен. Копать: slice-based encoding + ROI (центр приоритетнее), intra-refresh вместо полных keyframe, error concealment, FEC; настройки `mpph264enc` и CPE710 (ширина 20 МГц, ch 149, TX-power, DFS, distance), OpenIPC как референс. Мерить на живом движущемся видео.

### G (поле) — под вопросом
Power-cycle = грузится последний режим; Режим №1 + дрон — по решению пользователя.

## ЖЕЛЕЗО / ИСТИНА (locked) — *(источник: хэндофф 2026-06-13a + 2026-06-06, locked)*

- **u2** (drone, headless): зелёный провод → физ. **pin 26**, через SN74HC14N → ELRS-TX. **pin 26 = UART7_TX (m2) → `/dev/ttyS7`, TX-only, подтверждён.** RX/телеметрия = **m1 (29/38)**, резерв (трек D). Грабер Arkmicro `18ec:5555` → `/dev/video0`. IP: `10.8.0.7` (туннель) / `192.168.1.10` (мост).
- **u1** (оператор): CH340 `1a86:7523` → `/dev/ttyUSB0` (CRSF от RadioMaster Boxer, через SN74HC14N). HDMI-монитор. IP: `10.8.0.6` / `192.168.1.20`.
- **Бод линка: 420000** (везде — код, install.sh, baseline env, схемы). *(верифицировано этой сессией)*
- **CPE710**: AP `192.168.1.2`, Client `192.168.1.3`, ch 149, 20 МГц, TDMA off, distance 1 км.
- **Топология (locked):** CPE-AP → LAN PoE-адаптера → **мини-свитч** → {u1 + ПК}; u2 → LAN CPE-Client напрямую. **Мини-свитч на Конце A обязателен** (GMAC RK3588 не линкуется напрямую через пассивный PoE). Технику — только в LAN адаптера, **никогда** в POE/в сам CPE.
- **Каналы:** 1× ELRS-управление (TX-only) + 1× видео.
