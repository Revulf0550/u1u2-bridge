u1u2-bridge · field card

Смена режима · радиомост ⇄ WireGuard-туннель

Один переключатель: switch-mode.ps1 в корне репо. Запуск из обычного PowerShell. Скрипт не трогает физику и WG — их выставляешь руками до запуска.

u1 = RX + монитор · u2 = TX, headless сервисы u1: crsf-bridge@p1 · video-rx · wg-quick@wg0 сервисы u2: crsf-bridge@elrs · video-tx · wg-quick@wg0 u2 UART: /dev/ttyS7 = UART7 m2 → pin 26 (TX, одностор.)
Known-good узлы · 192.168.1.0/24 (статика — не меняется в любой сети; MAC зашит в железо)
IPMACузелгде
192.168.1.28c-86-dd-7a-31-b6CPE710 · AP (master)конец A
192.168.1.38c-86-dd-05-0a-6eCPE710 · Client (slave)конец B (за радио)

192.168.1.10c0-74-2b-fa-a4-ceu2-pi · TXконец B (за радио)
192.168.1.50—ПК · диагностика (любой)конец A, сбоку
Сверка в поле: 1..254 | %{ ping -n 1 -w 200 192.168.1.$_ | Out-Null }; arp -a -N <IP_ПК> | Select-String "динамич" — прогревает ARP и показывает все живые MAC разом.
⚠ Мерить серией, не одиночным пингом. ping -n 1/-n 2 по «холодному» ARP даёт ложные timeout — особенно для узлов за радиомостом (.3, .10): ARP-резолв через два CPE + воздух не успевает. Всегда ping -n 4+, либо пинг с самой Pi (ping -c 10). «Ответ от <IP_ПК>: Заданный узел недоступен» = это ПК сам себе говорит «MAC не нашёл», не ответ устройства. Не объявлять железо мёртвым по 1–2 пакетам.
Режим №1 · резерв
→ WireGuard-туннель (через интернет)
WireGuard ВКЛ Pi доступны по 10.8.0.6 / .7. WG нужен и для handshake, и для доступа.
  1. Физика: обе Pi → домашний свитч (нужен интернет для WG-handshake).
  2. WG на ПК: включить → Pi видны по 10.8.0.6/.7.
  3. Запуск (PowerShell, корень репо): 
    .\switch-mode.ps1 tunnel
  4. Подтвердить рестарт [y/N] — скрипт перезапустит crsf/video.
    дрон OFF на момент рестарта
  5. Проверка: 
    ping 10.8.0.7
ssh -i $HOME\.ssh\u1u2 ubuntu@10.8.0.7 "systemctl is-active crsf-bridge@elrs video-tx"
Режим №2 · боевой
→ радиомост CPE710 (прямой, по воздуху)
WireGuard ВЫКЛ kill-switch (0.0.0.0/0) иначе режет всю 192.168.1.x.
  1. Физика: Pi на концы CPE710. Конец A: u1 + ПК через мини-свитч (обязателен); u2 → CPE-Client.
  2. WG на ПК: выключить → Pi видны по 192.168.1.20/.10.
  3. Запуск (PowerShell, корень репо): 
    .\switch-mode.ps1 direct
  4. Подтвердить рестарт [y/N].
    дрон OFF на момент рестарта
  5. Проверка: 
    ping 192.168.1.20

Что важно помнить

  • Скрипт не трогает физику и WG — выставь их руками до запуска (см. гейт WG в каждой колонке).
  • -NoRestart — деплой конфига на диск без рестарта (безопасная проверка).
  • Конец A: мини-свитч обязателен — u1/RK3588 не линкуется напрямую в пассивный PoE CPE-AP. Технику только в порт LAN адаптера, никогда в POE / в сам CPE.
  • WG-ВКЛ и доступ к мосту 192.168.1.x — взаимоисключающие (kill-switch).
  • Скачал скрипт заново → Unblock-File .\switch-mode.ps1 (Mark-of-the-Web).
  • Под капотом: preflight-ping → git archive HEAD → scp → install.sh TRANSPORT=<mode> SKIP_APT=1 SKIP_NETPLAN=1 → рестарт.
  • Режим по умолчанию после power-cycle = последний задеплоенный (зашит в env на диске, авто-детекта нет).
  • video.env пишется из peer-адреса для обоих режимов — отдельный override не нужен.
